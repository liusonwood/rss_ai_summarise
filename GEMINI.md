# 🤖 GEMINI.md - RSS AI Daily Summarizer

This project is an automated RSS feed summarization tool powered by Python and OpenRouter (Gemini). It monitors an RSS source, summarizes new entries using AI, and publishes a new RSS feed containing these summaries.

## 🚀 Project Overview

- **Purpose**: To provide a condensed, AI-generated daily summary of RSS feeds to save time and reduce information overload.
- **Main Technologies**:
  - **Python 3**: Core processing logic using only standard libraries.
  - **OpenRouter API**: Integration with LLMs (defaulting to `google/gemini-2.0-flash-001`) for high-quality summarization.
  - **GitHub Actions**: Automation for daily execution and repository-based state persistence.
- **Architecture**:
  - **State Management**: `processed.txt` stores processed links to ensure incremental updates.
  - **Incremental Workflow**: `fetch` -> `filter` -> `clean` -> `summarize` -> `publish`.
  - **Git-as-DB**: The system treats the GitHub repository as a database, committing state changes back to the main branch after each run.

## 🛠 Building and Running

### Prerequisites
- Python 3.x
- An OpenRouter API Key.

### Running Locally
To test the script locally, set the required environment variables and run:
```bash
export OPENROUTER_API_KEY="your_key_here"
export RSS_SOURCE="path_to_local_xml_or_url"
export AI_MODEL="google/gemini-2.0-flash-001"
python3 summarize.py
```

### Automation
The project is configured to run automatically via GitHub Actions:
- **Schedule**: Daily at 12:00 UTC.
- **Manual**: Can be triggered via the "Actions" tab in the GitHub repository.

## 📝 Key Files

- `summarize.py`: The main entry point containing logic for fetching, cleaning, summarization, and XML generation.
- `.github/workflows/rss-summary.yml`: Defines the CI/CD pipeline for automated summarization.
- `processed.txt`: Persistent record of processed item IDs/links.
- `summary_feed.xml`: The output RSS feed generated by the script.
- `example-rss-feed`: A sample RSS file used for demonstration or default processing.
- `README.md`: User-facing documentation and setup guide.

## 📐 Development Conventions

- **Minimalism**: Favor Python standard libraries over third-party packages (e.g., `urllib.request` instead of `requests`).
- **Token Efficiency**: Always strip HTML and truncate content before sending it to the AI model.
- **Batching**: Summaries are generated by batching multiple items into a single prompt to save tokens and costs.
- **Security**: Never hardcode API keys; always use environment variables or GitHub Secrets.
- **Logical Comments**: Use "Why" comments to explain design decisions rather than "What" comments that repeat code logic.
